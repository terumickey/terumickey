# りそなインターン 空席通知ツール

りそなグループ インターンシップ マイページにログインし、
「RESONA Business Academy Premium(3daysワークショップ)」の日程選択ページを定期チェックして、
指定した日程の空席が出たらメールで通知します。

## 前提

- Node.js がインストールされていること
- 自分のマイページのID/パスワードを持っていること
- 対象コースへの応募・エントリーシート提出が完了していて、日程選択(申込内容を変更する)画面に進めること
- Gmailアカウント(2段階認証を有効にし、アプリパスワードを発行できること)

## 仕組み

1. マイページにログイン
2. トップページの「【3daysワークショップ】RESONA Business Academy Premium」の予約確認フォームを送信
3. 予約内容確認ページの「申込内容を変更する」フォームを送信 → 日程選択(空席状況)ページに到達
4. `.env` の `TARGET_DATES` に指定した日程が「満席」ラベル無しになっていれば「空席あり」と判定
5. 前回チェック時から `full → open` に変化した日程があればメール通知(`state.json` に前回状態を保存)

## セットアップ手順

### 1. 依存パッケージのインストール

```
npm install
```

(`postinstall` で Playwright 用の Chromium も自動ダウンロードされます)

### 2. `.env` ファイルの作成

`.env.example` をコピーして `.env` を作成し、中身を編集してください。

```
copy .env.example .env
```

**必ず `.env` を編集してください。`.env.example` はテンプレートなので実際の値を書き込まないでください。**

編集する項目:

- `RESONA_ID` / `RESONA_PASSWORD`: マイページのログイン情報
- `TARGET_DATES`: 空席通知の対象にしたい日程(カンマ区切り、複数可)。日程選択ページの表記(例: `2026/08/05`)に合わせる
- `GMAIL_USER`: 送信に使うGmailアドレス
- `GMAIL_APP_PASSWORD`: Googleアカウントの「アプリパスワード」(16桁、通常のログインパスワードは不可)
  - 発行方法: Googleアカウント管理 → セキュリティ → 2段階認証を有効化 → アプリパスワード
- `NOTIFY_TO`: 通知を受け取りたいメールアドレス

### 3. 動作確認

```
npm run check
```

コンソールに各日程ごとの `open` / `full` が表示されます。
`open` と判定され、かつ前回のチェックで `open` ではなかった日程がある場合のみメールが送信されます
(毎回送られるわけではなく、状態が変化した時だけ通知されます)。

状態は `state.json` に保存されます。

もし日程選択ページの構成が変わり画面遷移がうまくいかない場合は、以下のdiscoverモードで
ページの実際のテキスト・スクリーンショットを確認できます。

```
npm run discover
```

→ `day-select.txt` / `day-select.png` に保存されます。日程の表記や「満席」ラベルの位置が
変わっていないか確認し、必要なら `check.js` の `isDateFull` 関数を調整してください。

### 4. 定期実行の登録(Windows タスクスケジューラ)

PowerShellをこのフォルダで開いて実行:

```
powershell -ExecutionPolicy Bypass -File .\register-task.ps1
```

10分おきに `check.js` が自動実行されるようになります。
タスク名は `ResonaInternWatcher` です。`taskschd.msc` から確認・編集できます。

登録を削除したい場合:

```
powershell -ExecutionPolicy Bypass -File .\unregister-task.ps1
```

## 注意事項

- ID・パスワード・Gmailアプリパスワードは `.env` に平文で保存されます。他人と共有しているPCでは扱いに注意してください。
- `.env` と `state.json` は `.gitignore` 済みでGit管理対象外です。誤ってコミットしないよう注意してください。
- 自動ログイン・自動チェックはマイページの利用規約に抵触する可能性があります。ご自身の判断・責任で利用してください。
- チェック頻度は10分おきに設定していますが、`register-task.ps1` の `RepetitionInterval` を変更すれば調整できます。
- マイページの画面構成(フォーム名など)が変わると動かなくなる可能性があります。その場合は `npm run discover` で再調査してください。
